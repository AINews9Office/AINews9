The Phase 1 MVP is considered complete when users can:

- Start a Guided Learning Journey.
- Select an AI Journey.
- Select a Persona.

# EPIC 1
# Guided Learning Foundation

## Objective

Build Pragya's Guided Learning experience from the ground up.

This Epic transforms Pragya from a chatbot into a Guided AI Learning Companion.

---

## Business Outcome

Users should be able to:

- Start an AI learning journey.
- Select their AI journey.
- Select their persona.
- Ask AI questions.
- Receive guided responses.
- Continue learning.

---

## Sprint 1.1
### Guided Learning Experience

Capabilities

- Welcome Screen
- Guided Journey
- AI Journey Selection
- Persona Selection
- Topic Suggestions (Static MVP)
- Ask Your Own Question

Deliverable

Complete Guided Learning onboarding.

### Sprint 1.1 Implementation Scope

Sprint 1.1 should implement only the guided onboarding experience.

It should not implement the full Gateway, advanced Understanding Layer, persistent user history, authentication, analytics dashboard, or AI-generated topic suggestions.

Existing Pragya code should be reused and refactored where appropriate rather than replaced unnecessarily.

### Sprint 1.1 Technical Approach

- Use static configuration for AI journeys, personas, and topic suggestions.
- Store session context in memory for the MVP.
- Keep the Gateway minimal if touched during this sprint.
- Preserve compatibility with the existing FastAPI and widget foundation where practical.
- Avoid introducing a database in Sprint 1.1.
- Avoid LLM calls for onboarding option generation.

### Sprint 1.1 Acceptance Criteria

Sprint 1.1 is complete when:

- Pragya opens with a clear welcome experience.
- The user can select one of the four AI journeys: Learn AI, Use AI, Stay Safe, Grow with AI.
- The user can select one persona: Student, Teacher, Professional, Parent, Senior Citizen, Small Business, or Just Exploring AI.
- Pragya shows static topic suggestions based on the selected journey and persona.
- Pragya always provides an Ask Your Own Question option.
- The selected journey, persona, and topic or custom-question entry point are captured in session context.
- The user can enter the conversation after completing or bypassing topic selection through Ask Your Own Question.
- If suggestions are unavailable for a journey/persona pair, Pragya shows beginner-friendly fallback suggestions.
- The onboarding flow works without requiring authentication.
- The onboarding flow does not require a database.
- Existing chat functionality is not intentionally broken.

### Sprint 1.1 Edge Cases

Sprint 1.1 should handle:

- User opens Pragya and asks a question without choosing a suggested topic.
- User changes journey before selecting a persona.
- User changes persona before selecting a topic.
- User selects Just Exploring AI.
- User selects a journey/persona pair with no custom topic list.
- User refreshes the widget during an active in-memory session.

## Sprint 1.1 Capability Notes

### C-001 Welcome Experience

Acceptance criteria:

- Displays Pragya identity and AI Learning Companion positioning.
- Presents the guided journey as the primary path.
- Does not start with an empty chat-only experience.

### C-002 Guided Journey

Acceptance criteria:

- Guides the user through journey, persona, and topic selection.
- Allows the user to continue without technical knowledge.
- Keeps the flow simple and reversible within the current onboarding session.

### C-003 AI Journey Selection

Acceptance criteria:

- Supports exactly four Phase 1 journeys: Learn AI, Use AI, Stay Safe, Grow with AI.
- Stores the selected journey in session context.
- Uses the selected journey to filter topic suggestions.

### C-004 Persona Selection

Acceptance criteria:

- Supports Student, Teacher, Professional, Parent, Senior Citizen, Small Business, and Just Exploring AI.
- Stores the selected persona in session context.
- Uses the selected persona to personalize topic suggestions.

### C-005 Topic Suggestions

Acceptance criteria:

- Uses static configuration for Sprint 1.1.
- Shows suggestions based on selected journey and persona.
- Provides fallback suggestions when no exact match exists.
- Does not require an LLM call.

### C-006 Ask Your Own Question

Acceptance criteria:

- Remains available after journey and persona selection.
- Allows the user to enter the conversation without choosing a suggested topic.
- Preserves selected journey and persona context when the custom question is asked.

---

### C-021 AI News Service (Cloud Run)

Acceptance criteria:

- Migrates the AI News Feed backend from Render to Google Cloud Run.
- Provides a cloud-native deployment path using shared GCP infrastructure.
- Preserves the existing `/api/news` endpoint contract.
- Keeps the response schema backward compatible for the WordPress frontend.
- Supports independent scaling from Pragya's conversational backend.
- Uses Cloud Build for deployment automation.
- Uses Secret Manager for production secrets where required.
- Provides monitoring and logging suitable for production operations.
- Remains a platform capability, not a Pragya feature.
- Implementation scope is governed by `PRG-006_AI_News_Service_Specification.md`.

Implementation status:

- Implemented as an independent Cloud Run microservice under `AI_News_Service`.
- Production endpoint operational: `https://ai-news-service-888659608106.asia-south1.run.app`.
- `/health` and `/api/news` validated successfully on July 6, 2026.
- `/api/news` preserves the existing response contract and returns `{"news": [...]}`.
