# Before Writing Code

Ask yourself:

✓ Do I understand the requirement?

✓ Have I read the relevant documentation?

✓ Am I modifying the correct module?

✓ Am I preserving the architecture?

✓ Am I introducing unnecessary complexity?

If any answer is **No**, stop and investigate first.

---

# Engineering Principles

Always:

- Keep code simple.
- Reuse existing services.
- Preserve architecture.
- Write production-ready code.
- Prefer maintainability over cleverness.
- Keep documentation synchronized when architecture changes.

Never:

- Hardcode secrets.
- Duplicate business logic.
- Break existing APIs.
- Rewrite working modules unnecessarily.
- Ignore the Source of Truth hierarchy.
